# Логика НДС в калькуляторе YO

Документ описывает **как сейчас реализован** расчёт НДС в проекте: базовые формулы, Uzum, Wildberries (юнит и аналитика).  
Все суммы — **в узбекских сумах (UZS)**, если не указано иное.

Ставка по умолчанию: **12%** (поле `vatRate` / `wbVatPct` / `wbAnalyticsVatPct`).

---

## 1. Базовые функции (общие для Uzum и себестоимости)

Файл: `script.js` — функции `vatFromAmount`, `purchaseInputVat`.

### 1.1. НДС из суммы (`vatFromAmount`)

Используется для **исходящего НДС с цены продажи** и для ручного режима себестоимости в юнит-экономике.

| Режим | Поле в UI | Формула |
|--------|-----------|---------|
| `with` — сумма **включает** НДС | «Цена продажи: включает НДС» | `НДС = сумма × ставка / (100 + ставка)` |
| `without` — сумма **без** НДС | «Цена продажи: без НДС» | `НДС = сумма × ставка / 100` |

Пример при 12% и цене 112 000 сум «с НДС»:
`112 000 × 12 / 112 = 12 000 сум`.

### 1.2. Входящий НДС по закупке (`purchaseInputVat`)

Используется в **калькуляторе себестоимости** (ткань, пошив, упаковка и т.д.).

| Режим | Поле в UI | Формула |
|--------|-----------|---------|
| `with` — закупка **с НДС** | «да, сумма с НДС» | `входящий НДС = сумма × ставка / (100 + ставка)` |
| `without` — закупка **без НДС** | «нет, без НДС» | `входящий НДС = 0` |

Себестоимость без входящего НДС:
`нетто = брутто − входящий НДС`.

По каждой строке закупки считается `purchaseLine` / `purchaseLineFromValues`, затем суммируется **весь входящий НДС** (`totalInputVat`) и сохраняется в товар как `inputVat`.

**Хранение на складе** в себестоимость входит в брутто, но **без отдельного входящего НДС** (резерв хранения Uzum не облагается через `purchaseInputVat`).

## 2. Uzum — вкладка «Юнит-экономика»

Файл: `script.js` → `renderUnit()` (при маркетплейсе Uzum).

Схема: **исходящий НДС минус входящий НДС** (классическая «НДС к оплате»).

### 2.1. Входные поля

| Поле | ID | Назначение |
|------|-----|------------|
| Ставка НДС, % | `vatRate` | по умолчанию 12 |
| Цена продажи | `price` | цена на Uzum |
| Цена продажи с/без НДС | `saleVatMode` | `with` / `without` |
| Себестоимость | `cost` | брутто или нетто — зависит от `costVatMode` |
| Себестоимость с/без НДС | `costVatMode` | `with` / `without` |
| Привязка к товару | `useProductCost` | если включено — входящий НДС из базы |

### 2.2. Расчёт по шагам

```
комиссия_Uzum     = цена × комиссия% / 100
логистика         = по тарифу Uzum (литры / старый тариф)
хранение          = ставка × дни (или 0 по правилам SKU)

исходящий_НДС     = vatFromAmount(цена, ставка, saleVatMode)

входящий_НДС      = если привязан товар из базы:
                      state.linkedProduct.inputVat  (из калькулятора себестоимости)
                    иначе:
                      vatFromAmount(себестоимость, ставка, costVatMode)

НДС_к_оплате      = исходящий_НДС − входящий_НДС

полные_расходы    = себестоимость + комиссия + логистика + хранение + НДС_к_оплате

прибыль           = цена − полные_расходы
маржа             = прибыль / цена × 100%
```

### 2.3. Что показывается в UI

| Показатель | Элемент |
|------------|---------|
| Исходящий НДС | `outputVatVal` |
| Входящий НДС | `inputVatVal` |
| НДС к оплате | `vatPayableVal` |
| Источник входящего | `inputVatSourceVal` — «из себестоимости» или «ручной режим» |

### 2.4. Точка безубыточности

`calcBreakEven` использует ту же схему: на каждой пробной цене считается `vatPayable = outputVat − inputVat`, входящий НДС фиксирован.

## 3. Wildberries — вкладка «Юнит-экономика» (WB UZ)

Файл: `script.js` → `computeWbUnitEconomics()`, `renderWbUnitEconomics()`.

**Другая схема**, чем у Uzum: НДС **выделяется из суммы «к перечислению»**, входящий НДС по себестоимости **не вычитается**.

### 3.1. Сначала считается «к перечислению»

```
комиссия_WB   = цена × wbCommissionPct / 100
эквайринг     = цена × wbAcquiringPct / 100
реклама (ДРР) = цена × wbDrrPct / 100
логистика     = wbUzLogistics(литры, выкуп%, запас%, возврат фикс)
хранение      = литры × тариф × дни_оборачиваемости
приёмка       = wbAcceptanceUzs (если задано)

к_перечислению = цена − комиссия − эквайринг − реклама − логистика − хранение − приёмка
```

### 3.2. НДС (поле `wbVatPct`, по умолчанию 12%)

```
НДС = round(к_перечислению × ставка / (100 + ставка))   // если к_перечислению > 0
```

Это **не** схема «исходящий − входящий». НДС считается как доля внутри суммы вывода (аналог 12/112).

### 3.3. Прибыль

```
после_НДС     = к_перечислению − НДС
чистая_прибыль = после_НДС − себестоимость_ед (сум)
маржа          = чистая_прибыль / цена × 100%
ROI            = чистая_прибыль / себестоимость × 100%
```

### 3.4. UI

| Показатель | Элемент |
|------------|---------|
| К перечислению | `wbUnitToSellerVal` |
| НДС 12% (из суммы вывода) | `wbUnitVatVal` |
| Чистая прибыль | `wbUnitNetProfitVal` |

## 4. Uzum — аналитика продаж (отчёт Excel)

Файл: `script.js` → `computeUzumSalesAggregates()`.  
Вкладка: **Аналитика → Uzum**.

Ставка **жёстко 12%** (не берётся из `vatRate` юнит-экономики).

### 4.1. Данные по строкам отчёта

Для каждой строки продажи:

```
выручка_брутто   = колонка «Выручка»
выручка_нетто    = колонка «Выручка за вычетом комиссии» (или аналог)
услуги_Uzum      = выручка_брутто − выручка_нетто
продано_шт       = колонка «Продано»
COGS_строки      = из отчёта или 0
```

### 4.2. НДС — основная формула (для карточки «Итоговая чистая прибыль»)

По всем строкам накапливается:

```
исходящий_НДС_брутто += выручка_брутто × 12 / 112
НДС_с_услуг_Uzum     += услуги_Uzum × 12 / 112
входящий_НДС_COGS  += входящий_НДС_ед_из_БД × продано_шт
```

**Входящий НДС по товару из базы:** код ищет поле `totalIncomingVat` у товара.  
В текущей схеме сохранения товаров в БД пишется поле **`inputVat`**, а не `totalIncomingVat`.  
Если в базе нет `totalIncomingVat`, **входящий НДС по COGS в аналитике = 0** (фактическое поведение кода).

```
НДС_к_уплате = исходящий_НДС_брутто − НДС_с_услуг_Uzum − входящий_НДС_COGS

валовая_прибыль = выручка_нетто − COGS
чистая_прибыль  = валовая_прибыль − НДС_к_уплате    → показатель dashMetricCleanProfit
```

Смысл: из НДС с полной выручки вычитается НДС, «заложенный» в комиссию/услуги Uzum, и входящий НДС по закупке.

### 4.3. Альтернативный расчёт (в коде есть, в главной карточке не используется)

```
исходящий_нетто = выручка_нетто × 12 / 112
входящий_COGS   = COGS × 12 / 112
netProfit       = (выручка_нетто − исходящий_нетто) − (COGS − входящий_COGS)
```

Это упрощённая схема «нетто выручка и COGS без НДС». В UI на дашборд выводится **`cleanProfit`**, не `netProfit`.

### 4.4. Маржа в аналитике Uzum

```
маржа% = валовая_прибыль / общая_выручка_брутто × 100
```

(до вычета НДС, от брутто-выручки.)

## 5. Wildberries — аналитика еженедельного отчёта

Файл: `script.js` → `calculateWbReportAnalytics()`, настройки `readWbAnalyticsSettings()`.  
Вкладка: **Аналитика → WB**.

**Третья схема:** НДС = **процент от базы**, без выделения «внутри суммы» (12/112) и без входящего НДС по COGS.

### 5.1. Цепочка до прибыли

```
к_перечислению = SUM(payout по строкам «Продажа» и «Возврат»)
                 формула WB: розничная × (1 − кВВ%/100) − компенсация_платёжных

итого_к_оплате = к_перечислению
                 − логистика
                 − хранение
                 − удержания
                 − штрафы

COGS = себестоимость_ед × продано_шт (из базы товаров, сум)
```

### 5.2. База для НДС (настройка `vat_base`)

| Значение | UI | База |
|----------|-----|------|
| `payout` (по умолчанию) | «К перечислению (до логистики)» | `к_перечислению` |
| `after_direct_costs` | «Итого к оплате (после расходов WB)» | `итого_к_оплате` |

```
НДС = база × vat_rate        // vat_rate из поля wbAnalyticsVatPct, по умолчанию 0.12

чистая_прибыль = итого_к_оплате − COGS − НДС
маржа%         = чистая_прибыль / выручка_фактическая × 100%
```

Выручка фактическая — сумма колонки «Вайлдберриз реализовал Товар» по продажам (цена после СПП).

### 5.3. По артикулам (ABC-таблица)

На каждый SKU пропорционально выручке распределяются логистика, хранение, удержания:

```
доля = выручка_SKU / выручка_факт_итого

итого_к_оплате_SKU = payout_SKU − логистика×доля − хранение×доля − удержания×доля
база_НДС_SKU       = payout_SKU  или  итого_к_оплате_SKU  (по vat_base)
НДС_SKU            = база_НДС_SKU × ставка
прибыль_SKU        = итого_к_оплате_SKU − COGS_SKU − НДС_SKU
```

### 5.4. Что НЕ является НДС продавца

Колонка отчёта WB **«НДС с Вознаграждения Вайлдберриз»** — это налог **самого WB (РФ)**.  
В расчёт НДС продавца в Узбекистане **не входит** и в UI помечено как справочное.

## 6. Сводная таблица схем НДС

| Модуль | База / метод | Входящий НДС (закупка) | Формула НДС | Где вычитается из прибыли |
|--------|----------------|-------------------------|-------------|---------------------------|
| **Uzum юнит** | Цена продажи | Да (`inputVat` из себестоимости) | исходящий − входящий | В `totalCosts` как `vatPayable` |
| **WB юнит** | Сумма «к перечислению» | **Нет** | `к_перечислению × 12/112` | Отдельно от COGS: `afterVat − cost` |
| **Uzum аналитика** | Брутто-выручка | Должен быть `totalIncomingVat×шт` (см. §4.2) | брутто×12/112 − услуги×12/112 − входящий COGS | `cleanProfit = gross − vatToPay` |
| **WB аналитика** | `payout` или `after_direct_costs` | **Нет** | `база × ставка` (наценка, не 12/112) | `итого_к_оплате − COGS − НДС` |

## 7. Себестоимость → откуда берётся входящий НДС в других модулях

| Куда попадает | Поле в товаре | Как используется |
|---------------|---------------|------------------|
| Uzum юнит (привязка к товару) | `inputVat` | Подставляется как входящий НДС |
| WMS / поставки | `financialSnapshot.inputVat` | В колонке «Налог» детализации |
| Uzum аналитика | ожидается `totalIncomingVat` | **Сейчас не связано с `inputVat`** — см. §4.2 |
| WB юнит / WB аналитика | `costGross` / `costPriceUzs` | Только сумма закупки, входящий НДС **не вычитается** |

## 9. Важные отличия (чтобы не путать при сравнении)

1. **Uzum юнит** — классика: НДС к оплате = исходящий − входящий; входящий учитывается.
2. **WB юнит** — НДС как **доля внутри** суммы вывода (12/112), без входящего.
3. **WB аналитика** — НДС как **наценка на базу** (`× 0.12`), не 12/112; база выбирается в настройках.
4. **Uzum аналитика** — своя формула с вычетом НДС из услуг маркетплейса; ставка 12% зашита в код.
5. Раздел **«Финансы → Uzum»** (`finances.js`) **НДС не считает** — только выплаты, баланс ЛК, остатки.

## 10. Формулы одной строкой (шпаргалка)

**Uzum юнит:**
```
НДС_к_оплате = vat(цена) − vat(себестоимость или inputVat из базы)
прибыль = цена − себест − комиссия − логистика − хранение − НДС_к_оплате
```

**WB юнит:**
```
к_перечислению = цена − все_удержания_WB
НДС = к_перечислению × 12 / 112
прибыль = к_перечислению − НДС − себестоимость
```

**Uzum аналитика (cleanProfit):**
```
НДС_к_уплате = Σ(выручка×12/112) − Σ(услуги_Uzum×12/112) − Σ(inputVat_ед×шт)*
*если в БД есть totalIncomingVat; иначе последнее слагаемое = 0
чистая = (нетто_выручка − COGS) − НДС_к_уплате
```

**WB аналитика:**
```
итого_к_оплате = к_перечислению − логистика − хранение − удержания − штрафы
НДС = (к_перечислению ИЛИ итого_к_оплате) × ставка
чистая = итого_к_оплате − COGS − НДС
```
